Project Proposal:
Mood Based Weather App

Description

The project will be to develop a weather app that will not only provide you with weather details from a 3rd party source, but also collect information about the user and their mood on the given day and how the weather affects them. The app will take the information from its users and correlate how that particular weather makes people feel based on temperature, forecast and humidity. It will let the user know what kind of mood the weather will likely put them in on a given day. Additionally, it will also seek to ask how productive a person feels against that weather to in future provide insight to how weather may affect their productivity.

Components to the project:

  1. Weather Data Integration: The app will pull live weather data using a public weather API (e.g., OpenWeatherMap, Weatherstack) to display real-time weather conditions.
  2. Mood Logging: Each day, users will be prompted to log their mood in the app, with options like “Happy,” “Sad,” “Energetic,” or “Relaxed.” Users can add short text entries to describe how the weather makes them feel.
  3. Data Storage and Analytics: Azure Functions will handle the backend processing. Mood logs and weather data will be stored in Azure Cosmos DB. The app will track correlations between weather conditions (temperature, humidity, wind) and user mood.
  4. Personalised Insights: Over time, the app will analyse patterns and offer insights like “You tend to feel more productive on sunny days” or “Cloudy weather makes you feel more relaxed.”
  5. Sentiment Analysis: To enhance user experience, Azure Cognitive Services can be used for sentiment analysis of mood logs, identifying emotional trends.
